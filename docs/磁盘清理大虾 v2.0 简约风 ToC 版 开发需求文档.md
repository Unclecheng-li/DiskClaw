# 磁盘清理大虾 v2\.0 简约风 ToC 版 开发需求文档

**文档版本**：v1\.0
**编写日期**：2026\-05\-18
**目标平台**：Windows 10/11（优先）、macOS
**当前版本**：v1\.0 技术原型版

## 一、项目背景与核心升级目标

### 1\.1 问题诊断

当前 v1\.0 版本是典型的**开发者工具风格**，存在以下致命的 ToC 体验问题：

- 信息过载：单页展示所有功能（配置、日志、报告、执行），普通用户完全不知所措

- 专业术语泛滥：\&\#34;Dry Run\&\#34;、\&\#34;热点目录\&\#34;、\&\#34;报告索引\&\#34; 等术语门槛极高

- 视觉风格老旧：深绿色调、密集排版、生硬的卡片设计，缺乏现代感和亲和力

- 核心流程复杂：用户需要先配置 LLM、设置参数、生成方案才能开始清理，步骤超过 5 步

- 功能优先级倒置：系统日志、配置参数等开发者功能占据了最显眼的位置

### 1\.2 核心升级目标

1. **彻底重构为 ToC 产品**：让普通电脑用户无需任何学习成本就能使用

2. **极简核心流程**：打开软件→一键扫描→一键清理，3 步完成所有操作

3. **现代简约设计**：采用当前主流的清新简约风格，大留白、柔和阴影、流畅动画

4. **功能分层设计**：核心功能前置，高级功能隐藏到二级页面，兼顾普通用户和高级用户

5. **保留技术内核**：不改变原有 LLM 智能分析、文件隔离、报告生成等核心技术能力

6. **最小化技术变更**：完全基于现有技术栈实现，不引入任何新的前端框架或依赖

## 二、现有技术栈确认（无变更）

本次升级**完全复用现有技术栈**，不引入任何新的第三方框架或库：

|技术领域|具体实现|入口文件 / 配置|
|---|---|---|
|运行时|Node\.js ES Modules \(`\&\#34;type\&\#34;: \&\#34;module\&\#34;`\)|`package\.json`|
|桌面端|Electron ^37\.2\.6|`electron/main\.js`|
|后端服务|原生 `node:http` 自建 HTTP 服务|`src/startServer\.js`、`src/routes\.js`|
|前端界面|原生 HTML \+ CSS \+ Vanilla JavaScript|`src/ui/dashboard\.js`|
|LLM 接入|OpenAI\-compatible API（原生 `fetch`）|`src/services/llm/openAiCompatibleClient\.js`|
|本地存储|JSON 文件 \+ 文件系统目录（`\.diskclaw`）|`src/config/defaults\.js`|
|加密|`node:crypto` AES\-256\-GCM|`src/utils/crypto\.js`|
|打包发布|electron\-builder ^26\.8\.1|`electron\-builder\.yml`|
|测试|自定义 Node 测试脚本 \+ `node:assert/strict`|`scripts/run\-tests\.js`|

## 四、各页面详细设计需求

### 4\.1 首页（核心功能页）\- 最重要页面

**设计原则**：极致简约，只有一个核心动作，没有任何多余信息

#### 4\.1\.1 初始状态（未扫描）

- 页面中央显示一个**超大圆形扫描按钮**（直径≥200px），按钮内文字 \&\#34;一键智能扫描\&\#34;

- 按钮上方显示应用 Logo 和 \&\#34;磁盘清理大虾\&\#34; 标题

- 按钮下方显示一行小字：\&\#34;AI 智能分析您的磁盘，安全清理垃圾文件\&\#34;

- 页面底部显示 \&\#34;上次清理：XX 天前，已释放 XXGB 空间\&\#34;（如果有历史记录）

- 整个页面除了侧边栏导航，没有任何其他元素，大面积留白

#### 4\.1\.2 扫描中状态

- 圆形按钮变为扫描动画状态（旋转的进度环）

- 按钮内文字变为 \&\#34;正在扫描中\.\.\.\&\#34;

- 按钮下方实时显示：

    - 已扫描文件数

    - 已发现可清理空间

    - 当前正在扫描的路径（滚动显示）

- 显示 \&\#34;取消扫描\&\#34; 按钮（次要按钮，灰色）

#### 4\.1\.3 扫描完成状态

- 圆形按钮变为绿色对勾图标

- 按钮上方用**超大字体**显示 \&\#34;可清理空间：XXX MB/GB\&\#34;

- 下方显示垃圾文件分类卡片（横向排列，最多 4 个）：

    - 系统垃圾

    - 重复文件

    - 下载文件

    - 不常用文件

- 每个卡片显示该分类的可清理空间大小

- 最下方显示**超大橙色 \&\#34;一键清理\&\#34; 按钮**（主按钮，最突出）

- 清理按钮下方显示小字：\&\#34;所有文件将先移动到隔离区，可随时恢复\&\#34;

#### 4\.1\.4 清理中状态

- 显示清理进度条

- 实时显示：已清理 XX 个文件，已释放 XX 空间

- 显示 \&\#34;取消清理\&\#34; 按钮

#### 4\.1\.5 清理完成状态

- 显示庆祝动画（彩色纸屑或绿色对勾）

- 用超大字体显示 \&\#34;清理完成！\&\#34;

- 下方显示 \&\#34;本次共释放 XX 空间，清理了 XX 个文件\&\#34;

- 显示 \&\#34;查看详情\&\#34; 和 \&\#34;返回首页\&\#34; 两个按钮

### 4\.2 隔离区页面

- 顶部显示 \&\#34;隔离区\&\#34; 标题和 \&\#34;清空隔离区\&\#34; 按钮（红色危险按钮）

- 下方显示文件列表，每行包含：

    - 文件名称

    - 原路径（截断显示，鼠标悬停显示完整路径）

    - 隔离时间

    - 文件大小

    - 操作按钮：\&\#34;恢复\&\#34;、\&\#34;永久删除\&\#34;

- 支持批量选择文件，批量恢复 / 删除

- 顶部显示统计信息：\&\#34;共隔离 XX 个文件，占用 XX 空间\&\#34;

- 显示自动清理提示：\&\#34;隔离文件将在 30 天后自动永久删除\&\#34;

### 4\.3 清理历史页面

- 顶部显示 \&\#34;清理历史\&\#34; 标题

- 按时间倒序显示历史清理记录卡片

- 每个卡片包含：

    - 清理时间

    - 清理文件数

    - 释放空间

    - 操作按钮：\&\#34;查看报告\&\#34;、\&\#34;导出报告\&\#34;

- 点击 \&\#34;查看报告\&\#34; 弹出模态框，显示详细的清理报告

- 支持按时间范围筛选历史记录

### 4\.4 设置页面

**设计原则**：分类清晰，默认隐藏高级选项，普通用户无需进入

#### 4\.4\.1 基础设置（默认展开）

- 自动扫描：开启 / 关闭，设置扫描间隔（每天 / 每周 / 每月）

- 自动清理：开启 / 关闭，设置清理时间（扫描完成后自动清理）

- 隔离文件保留时间：7 天 / 15 天 / 30 天 / 永久

- 通知设置：扫描完成通知、清理完成通知、错误通知

#### 4\.4\.2 LLM 配置（默认折叠）

- 完全保留原 v1\.0 的 LLM 配置功能

- 添加 \&\#34;使用默认配置\&\#34; 按钮，一键填充推荐值

- 添加 \&\#34;测试连接\&\#34; 按钮

- 添加提示文字：\&\#34;修改此配置可能影响 AI 分析准确性，非必要请勿修改\&\#34;

#### 4\.4\.3 规则配置（默认折叠）

- 完全保留原 v1\.0 的扫描参数和黑白名单配置

- 添加 \&\#34;恢复默认规则\&\#34; 按钮

- 添加提示文字：\&\#34;修改此配置可能导致误删重要文件，请谨慎操作\&\#34;

#### 4\.4\.4 系统设置（默认展开）

- 开机自动启动：开启 / 关闭

- 关闭窗口时：最小化到托盘 / 退出应用

- 窗口大小：记住上次大小 / 固定大小

- 语言：简体中文 / English

- 检查更新：自动检查 / 手动检查

#### 4\.4\.5 高级设置（默认折叠）

- 系统日志：查看日志、导出日志、清空日志

- 数据管理：备份配置、恢复配置、重置所有设置

- 调试模式：开启 / 关闭

## 五、视觉设计规范

### 5\.1 整体风格

- **风格定位**：现代简约、清新科技、安全可靠

- **设计语言**：大圆角、柔和阴影、渐变色彩、流畅动画、大面积留白

- **布局原则**：内容居中，左右对称，呼吸感充足，避免密集排版

### 5\.2 颜色系统

|颜色类型|色值|使用场景|
|---|---|---|
|主色调|\#10B981（翡翠绿）|主按钮、图标、强调文字|
|辅助色 1|\#3B82F6（蓝色）|链接、次要按钮、信息提示|
|辅助色 2|\#F59E0B（橙色）|警告提示、清理按钮|
|危险色|\#EF4444（红色）|删除按钮、错误提示|
|中性色 1|\#F9FAFB|页面背景|
|中性色 2|\#F3F4F6|卡片背景、输入框背景|
|中性色 3|\#9CA3AF|次要文字、提示文字|
|中性色 4|\#1F2937|主要文字、标题|

### 5\.3 字体系统

- 标题字体：系统无衬线字体（Segoe UI / San Francisco），字重 600\-700

- 正文字体：系统无衬线字体，字重 400

- 字号层级：

    - 大标题：32px\-48px

    - 页面标题：24px

    - 卡片标题：18px

    - 正文：14px\-16px

    - 提示文字：12px\-13px

### 5\.4 组件规范

- 圆角：所有组件统一使用 12px 圆角（大按钮 16px，圆形按钮 50%）

- 阴影：使用柔和的 box\-shadow，避免生硬的边框

    - 卡片阴影：`0 2px 8px rgba\(0, 0, 0, 0\.08\)`

    - 按钮阴影：`0 4px 12px rgba\(16, 185, 129, 0\.3\)`

- 间距：使用 8px 网格系统，所有间距为 8px 的倍数

- 动画：所有交互添加 0\.2s\-0\.3s 的过渡动画，包括悬停、点击、页面切换

### 5\.5 侧边栏设计

- 宽度：240px

- 背景色：白色，轻微阴影

- 导航项：图标 \+ 文字，选中状态使用主色调背景和白色文字

- 底部显示应用版本号和 \&\#34;检查更新\&\#34; 链接

## 六、交互与体验优化

### 6\.1 核心流程优化

- 应用启动后直接进入首页，自动聚焦 \&\#34;一键扫描\&\#34; 按钮

- 自动更新：支持静默自动更新，更新完成后提示用户重启

## 七、技术实现要点（基于原生技术栈）

### 7\.1 前端架构调整（无框架）

- **单页多视图实现**：在 `src/ui/dashboard\.js` 中新增视图切换逻辑

    - 定义 `views` 对象，包含所有页面的 HTML 模板字符串

    - 实现 `switchView\(viewName\)` 函数，替换主内容区域的 HTML

    - 侧边栏导航点击时调用 `switchView` 切换页面

- **状态管理**：使用全局 `appState` 对象管理应用状态

    ```javascript
    const appState = {
      currentView: 'home',
      scanStatus: 'idle', // idle/scanning/complete
      scanProgress: 0,
      scanResults: null,
      // 其他全局状态
    };
    ```

- **事件委托**：使用事件委托处理所有动态生成元素的点击事件，避免重复绑定

- **API 通信**：继续使用原生 `fetch` 调用后端 HTTP 接口，保持与现有代码一致

### 7\.2 性能优化

- **按需渲染**：只渲染当前可见的页面内容，隐藏其他页面

- **扫描进度节流**：每 500ms 更新一次 UI，避免频繁重绘

- **虚拟滚动**：隔离区和清理历史的长列表使用原生 JS 实现虚拟滚动

- **日志增量加载**：系统日志默认只加载最新 50 条，滚动到底部时加载更多

### 7\.3 动画实现

- **CSS 动画**：所有基础动画（悬停、过渡、旋转）使用纯 CSS 实现

- **扫描进度环**：使用 SVG \+ CSS `stroke\-dasharray` 实现环形进度条

- **页面切换动画**：使用 CSS `opacity` 和 `transform` 实现淡入淡出效果

- **庆祝动画**：使用原生 JS 生成彩色纸屑元素，添加下落动画

### 7\.4 后端与主进程调整

- **后端接口保持不变**：所有现有 API 端点完全复用，不做任何修改

- **主进程通信保持不变**：继续使用 Electron IPC 机制实现主进程与渲染进程通信

- **数据格式保持不变**：所有 JSON 数据格式与 v1\.0 完全兼容

- **打包配置保持不变**：`electron\-builder\.yml` 配置无需修改
