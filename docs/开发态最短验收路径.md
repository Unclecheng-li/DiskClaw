# 磁盘清理大虾 开发态最短验收路径

日期：2026-05-16  
适用场景：`npm run electron`

## 目标

用最短步骤确认开发态页面已经恢复正常交互，并验证最近新增功能没有再次把前端脚本打坏。

## 步骤

### 1. 启动

执行：

```bash
npm run electron
```

确认窗口正常打开。

### 2. 切换标签页

依次点击：

- `总览`
- `设置`
- `执行与报告`

确认页面能正常切换，没有卡死。

### 3. 设置页最短验证

在“设置”页依次点击：

- `读取已保存配置`
- `恢复默认输入`
- `读取规则`
- `读取偏好`
- `读取计划`

只要这些按钮都能有反馈，说明开发态基础交互已经恢复。

### 4. 总览页最短验证

回到“总览”页，点击：

- `填充示例配置`
- `仅扫描`

确认顶部任务状态条开始变化。

### 5. 方案与执行最短验证

点击：

- `生成清理方案`

等方案完成后，再点击优先事项，确认能自动跳到“执行与报告”页并触发对应 Dry Run。

### 6. 报告与任务最短验证

在“执行与报告”页点击：

- `加载报告索引`
- `加载任务列表`

再点击任意一条任务或报告，确认详情区域能更新。

## 结论判断

如果以上 6 步都正常，基本可以判断：

- 开发态页面脚本已恢复
- 核心按钮交互正常
- 页面初始化不会再因为脚本错误整体失效
