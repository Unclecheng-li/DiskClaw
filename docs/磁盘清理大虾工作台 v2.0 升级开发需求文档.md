# 磁盘清理大虾工作台 v2\.0 升级开发需求文档

**文档版本**：v1\.0
**编写日期**：2026\-05\-16
**目标平台**：Windows 10/11（优先）、macOS、Linux
**技术栈**：Electron 30\.x \+ Vue 3 \+ Tailwind CSS \+ Node\.js 20\.x
**当前版本**：v1\.0 原型版

## 一、项目背景与升级目标

### 1\.1 项目背景

磁盘清理大虾工作台是一款基于 LLM 大模型的智能桌面磁盘清理工具，通过 AI 分析用户磁盘文件的使用频率、重要性和风险等级，提供个性化的清理建议。当前 v1\.0 版本已实现核心功能流程（模型接入→扫描规划→模拟清理→实际执行→报告生成→文件隔离），但存在用户体验差、安全机制缺失、功能不完善、视觉不统一等问题，严重影响工具的可用性和推广价值。

### 1\.2 升级目标

1. **安全性提升**：建立完善的文件操作安全防护体系，杜绝误删和数据丢失风险

2. **体验优化**：大幅降低普通用户的使用门槛，消除专业术语障碍

3. **功能完善**：补齐核心功能缺失，增强 Electron 桌面应用原生特性

4. **性能优化**：解决大日志加载卡顿、扫描无响应等性能问题

5. **视觉统一**：建立完整的设计规范，提升界面专业度和一致性

## 二、总体需求概述

本次升级将围绕 \&\#34;安全第一、体验优先\&\#34; 的原则，对 v1\.0 版本进行全面重构和优化。重点解决系统日志失控、危险操作无防护、核心交互缺失三大核心问题，同时充分利用 Electron 框架的桌面特性，打造一款专业、易用、安全的智能磁盘清理工具。

## 三、详细功能需求

### 3\.1 核心安全与交互升级（P0 最高优先级）

#### 3\.1\.1 危险操作二次确认机制

- 所有会修改磁盘文件的操作（执行普通清理、执行重复文件隔离、执行热点目录隔离、永久删除隔离文件）必须添加二次确认对话框

- 确认对话框需明确显示操作影响范围（如 \&\#34;将永久删除 3 个文件，释放 466\.8MB 空间\&\#34;）

- 提供 \&\#34;不再提示\&\#34; 选项，但仅针对同一类型的低风险操作

- 永久删除操作需额外添加 \&\#34;输入确认文字\&\#34; 的强验证机制

#### 3\.1\.2 全流程操作进度反馈

- 扫描过程：显示实时扫描进度条、当前扫描路径、已扫描文件数、已发现可清理文件数

- 清理过程：显示清理进度条、当前处理文件、已清理文件数、已释放空间

- 隔离过程：显示隔离进度条、当前隔离文件、已隔离文件数

- 所有长时间操作（\&gt;3 秒）必须提供取消按钮

- 操作完成后显示结果摘要（成功数、失败数、释放空间）

#### 3\.1\.3 全局错误提示系统

- 建立统一的错误提示机制，替代当前仅在系统日志中显示错误的方式

- 错误提示分为三个级别：

    - 成功提示（绿色）：操作成功的确认信息

    - 警告提示（橙色）：非致命错误，不影响主流程

    - 错误提示（红色）：致命错误，操作失败

- 所有错误提示需包含用户可理解的错误描述和建议解决方案

- 提供 \&\#34;查看详细日志\&\#34; 按钮，点击后跳转到系统日志并定位到对应错误行

### 3\.2 系统日志模块重构（P0 最高优先级）

#### 3\.2\.1 日志展示优化

- 默认只显示最新 5 条日志摘要，隐藏完整 JSON 内容

- 添加 \&\#34;展开全部\&\#34; 和 \&\#34;收起\&\#34; 按钮，控制完整日志的显示

- 实现 JSON 语法高亮，提升开发者可读性

- 日志按级别区分颜色：INFO（灰色）、WARN（橙色）、ERROR（红色）、DEBUG（蓝色）

#### 3\.2\.2 日志交互功能

- 添加日志搜索功能，支持按关键词、级别、时间范围过滤

- 添加日志级别筛选器（全部、INFO、WARN、ERROR、DEBUG）

- 添加 \&\#34;清空日志\&\#34; 按钮，带二次确认

- 添加 \&\#34;导出日志\&\#34; 按钮，支持导出为 TXT 和 JSON 格式

#### 3\.2\.3 隐私与安全优化

- 对日志中的用户敏感信息进行自动脱敏处理（用户名、完整文件路径中的个人信息）

- 日志文件自动轮转，单个日志文件大小不超过 10MB，最多保留 7 个历史日志文件

- 日志存储在 Electron 应用的 userData 目录下，避免暴露在公共位置

### 3\.3 核心功能完善（P1 高优先级）

#### 3\.3\.1 文件选择与路径输入优化

- 为扫描目录输入框添加 \&\#34;浏览文件夹\&\#34; 按钮，调用系统原生文件选择对话框

- 支持拖拽文件夹到输入框快速添加扫描路径

- 支持同时添加多个扫描目录，显示为标签列表，可单独删除

- 保存用户历史扫描路径，提供下拉选择

#### 3\.3\.2 隔离区功能增强

- 支持单个文件恢复和批量恢复

- 支持单个文件永久删除和批量永久删除

- 显示每个隔离文件的原路径、隔离时间、文件大小

- 添加 \&\#34;清空隔离区\&\#34; 按钮，带二次确认

- 隔离文件自动过期机制（默认 30 天，可在设置中修改）

#### 3\.3\.3 报告与任务管理优化

- 支持选择任意历史报告进行导出（JSON 和 Markdown 格式）

- 实现最近任务与任务详情的联动：点击最近任务列表中的任务，自动更新任务详情区域

## 五、技术实现要点

### 5\.1 Electron 主进程与渲染进程通信

- 使用 IPC 机制实现主进程与渲染进程的安全通信

- 所有文件系统操作（扫描、清理、隔离）在主进程中执行

- 渲染进程只负责 UI 展示和用户交互，不直接操作文件系统

### 5\.2 文件系统操作

- 使用 Node\.js 的 fs 模块和 glob 库进行文件扫描

- 使用 stream 方式处理大文件，避免内存溢出

- 实现文件操作的原子性，确保在断电或程序崩溃时不会损坏用户文件

### 5\.3 日志系统

- 使用 winston 库实现日志的记录、轮转和分级

- 前端使用 jsonlint 和 highlight\.js 实现 JSON 语法高亮

- 实现日志的增量加载，避免一次性加载大量日志导致页面卡顿

### 5\.4 UI 框架

- 使用 Tailwind CSS v3 实现样式统一和响应式设计

- 使用 Element Plus 组件库提供基础 UI 组件

- 自定义主题色，匹配应用整体风格
